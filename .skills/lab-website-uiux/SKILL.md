# Academic Lab Website UI/UX Skill

You are designing a formal academic laboratory website for a university professor and research group.

## Design Goal

Create a site that feels:

- academic
- trustworthy
- precise
- refined
- modern
- research-oriented
- suitable for a National Cheng Kung University mechanical engineering materials laboratory

It must not feel like:

- a generic template
- a startup landing page
- a student side project
- a cartoon technology website
- an over-animated AI-generated website

## Lab-Specific Identity

This lab focuses on:

- green synthesis
- circular economy
- nanomaterials
- biopolymer composites
- surface engineering
- tribology
- biomedical materials
- environmental materials
- sensing materials
- simulation-assisted material and process design

Use these themes as the visual basis.

Preferred visual motifs:

- molecular lattice
- microstructure texture
- layered material cross-section
- particle network
- subtle contour/grid pattern
- fiber-like material lines
- sensor signal waves
- coating layers

Avoid random icons unrelated to materials science.

## Visual System

Use:

- off-white background
- deep navy or muted blue
- graphite text
- restrained green/gold accents
- subtle borders
- soft shadows
- generous spacing
- clean typography
- consistent max-width container

Avoid:

- neon gradients
- too many colors
- low-contrast gray text
- inconsistent cards
- cramped layouts
- excessive animations
- stock photos pretending to be the actual lab

## Required Pages

- Home
- About
- Research
- Members
- Publications
- Facilities
- News
- Join Us
- Contact

## Animation Rules

Use subtle, slow, content-supporting animations.

Recommended:

- fade + translate entrance
- scroll reveal
- card hover lift
- sticky header background transition
- slow moving gradient
- low-opacity particle/lattice motion
- mobile menu transition
- publication hover highlight

Required accessibility:

- support `prefers-reduced-motion`
- no essential information conveyed only by animation
- no aggressive parallax
- no flashing
- no distracting infinite motion near reading text

## Typography

Use readable academic typography:

- body size 16–18px
- line height 1.65–1.8
- strong H1/H2 hierarchy
- no tiny captions unless secondary
- Chinese text should remain readable and balanced
- avoid all caps except small labels

## Layout Rules

Check:

- consistent section padding
- consistent container width
- aligned card grids
- equal card heights where appropriate
- readable text width
- mobile stacking
- no horizontal scroll
- header not covering content
- footer not cramped

## Content Rules

Do not invent private or unverified information.

Use placeholders for:

- members
- equipment
- lab photos
- logo
- real publications
- awards
- grants

Placeholders should look intentional and polished.

## Visual QA Process

Before final answer:

1. Run the project.
2. Inspect in browser or Playwright.
3. Check at least:
   - 1440px desktop
   - 1024px tablet
   - 768px tablet
   - 390px mobile
4. Fix visual issues:
   - spacing
   - alignment
   - overflow
   - text wrapping
   - mobile nav
   - card consistency
   - contrast
5. Repeat until the UI is polished.

## Final Response Requirements

When done, report:

- files changed
- pages implemented
- design system choices
- animations/interactions implemented
- how to run locally
- visual QA performed
- screenshots generated
- known placeholders
